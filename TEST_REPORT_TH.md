# รายงานตรวจสอบ Rendo PWA v1.1.1

วันที่ตรวจสอบ: 17 กรกฎาคม 2569

## สาเหตุที่พบจริง

หน้าลงทำงานค้างไม่ได้เกิดจาก Firebase โดยตรง แต่เกิดจากฟังก์ชันเพิ่มวันที่เดิมใช้วันที่เวลาแบบท้องถิ่น แล้วแปลงกลับด้วย `toISOString()` ซึ่งเป็นเวลา UTC

ในเขตเวลาไทย ตัวอย่างวันที่ `2026-07-01` เมื่อเพิ่ม 1 วันแล้วแปลงกลับแบบเดิม อาจได้ `2026-07-01` เหมือนเดิม ทำให้ลูปตรวจวันที่ขาดไม่เดินหน้าและวนไม่จบ จึงบล็อก JavaScript ทั้งหน้า ผู้ใช้เลยกดหรือเลื่อนอะไรไม่ได้จนต้องปิดแอป

## สิ่งที่แก้ใน v1.1.1

- แก้การเพิ่มวันให้คำนวณด้วย UTC แบบกำหนดปี เดือน วันโดยตรง
- เปลี่ยนการหาวันที่ยังไม่ได้ลงเป็นลูปเลขวันที่ 1–31 แบบมีขอบเขตตายตัว
- แยกตัวนับการโหลดฟอร์มวันทำงานกับตัวนับรายการรายเดือน ไม่ใช้ตัวเดียวกัน
- ระหว่างโหลดข้อมูลเดิม ปิดเฉพาะปุ่มบันทึก ไม่ปิดการกดทั้งฟอร์มและไม่ปิดเมนู
- ป้องกันข้อมูลโหลดช้าจากหน้าเดิมกลับมาทับหน้าที่ผู้ใช้เปิดใหม่
- เพิ่มปุ่มลองใหม่และข้อความผิดพลาดเฉพาะส่วน แทนการปล่อยให้หน้าค้าง
- ปรับช่องตัวเลขหน้ายอดขาย: แตะเลข 0 แล้วพิมพ์เลขใหม่ได้ทันที และเมื่อปล่อยช่องว่างจะคืนเป็น 0
- เงินเปิดกะค้นจากยอดปิดกะของวันเปิดร้านก่อนหน้า โดยข้ามวันที่หยุดและค้นย้อนหลังได้สูงสุด 400 รายการ
- เพิ่ม cache ระยะสั้น 10 วินาทีสำหรับข้อมูลรายเดือน และล้าง cache ทันทีเมื่อมีการบันทึก
- เพิ่มการป้องกันผลโหลดเก่าใน Dashboard, เบิกล่วงหน้า, รายจ่าย, ประวัติ, ยอดขาย, รายเดือน, ค่าตอบแทน และลงทำงาน
- อัปเดต Service Worker เป็น build `1.1.1`

## การตรวจสอบที่ผ่าน

- `node --check app.js` — ผ่าน
- `node --check sw.js` — ผ่าน
- ตรวจ JSON ของ `manifest.webmanifest`, `version.json`, `firebase.json`, `.firebaserc` และ `firestore.indexes.json` — ผ่าน
- ทดสอบ Browser จำลองด้วย Chromium ในเขตเวลา `Asia/Bangkok` — ผ่าน
- ตรวจ `addDaysISO('2026-07-01', 1)` ได้ `2026-07-02` — ผ่าน
- ตรวจเดือนมิถุนายนที่ไม่มีข้อมูลคืนวันที่ขาด 30 วันและจบการทำงาน — ผ่าน
- เปิดหน้าลงทำงาน เปลี่ยนสถานะ และกดโหลดซ้ำโดยหน้าไม่ค้าง — ผ่าน
- สลับหน้าเร็วระหว่าง Dashboard, เบิกล่วงหน้า, รายจ่าย, ประวัติ, รายเดือน และยอดขาย โดยหน้าสุดท้ายไม่ถูกหน้าเก่าทับ — ผ่าน
- ช่องยอดขายเริ่ม 0 แล้วพิมพ์ `500` ได้ค่า `500` ไม่ใช่ `0500` — ผ่าน
- ลบค่าช่องตัวเลขจนว่างแล้วออกจากช่อง ระบบคืนเป็น 0 — ผ่าน
- เงินเปิดกะดึงจากเงินปิดกะวันก่อนหน้าจำลองได้ 1,234 บาท — ผ่าน
- Browser test ไม่พบ `pageerror` — ผ่าน

## ข้อจำกัดของการทดสอบ

การทดสอบ Browser ใช้ข้อมูล Firebase จำลองเพื่อทดสอบตรรกะหน้าเว็บ ความค้าง การสลับหน้า และการกรอกข้อมูล ยังไม่ได้เชื่อมต่อบัญชี Firebase, Google Drive และ LINE จริงของร้าน จึงควรทดสอบบนระบบจริงก่อนใช้จ่ายเงินจริง

## Checklist หลังอัปโหลด

1. เปิดแอปและตรวจมุมบนว่าเป็น `v1.1.1`
2. Login ด้วยเจ้าของและพนักงานอย่างน้อย 2 ระดับ
3. เข้า **ลงทำงาน** แล้วเปลี่ยนวันที่ 3–5 ครั้งติดกัน
4. เปลี่ยนหน้า **ลงทำงาน → รายเดือน → ยอดขาย** อย่างรวดเร็ว
5. กรอกยอดขาย 500 ในช่องที่เริ่มจาก 0
6. เลือกวันที่ใหม่ที่ยังไม่มียอดขาย และตรวจเงินเปิดกะ
7. บันทึกวันทำงานจากมือถือ 2 เครื่องแล้วเปิดรายเดือนตรวจผล
8. สร้าง Backup JSON ก่อนเริ่มใช้จริง

ไม่สามารถรับประกันว่าแอปที่พึ่งพาอินเทอร์เน็ตและบริการภายนอกจะไม่มีข้อผิดพลาดทุกกรณี แต่เวอร์ชันนี้แก้ลูปค้างที่พบจริง เพิ่ม timeout การยกเลิกผลโหลดเก่า และการคืนสถานะปุ่มเพื่อให้ไม่ต้องปิดแอปเมื่อเครือข่ายช้าหรือผู้ใช้สลับหน้าเร็ว
