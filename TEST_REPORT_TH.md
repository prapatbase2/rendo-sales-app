# รายงานตรวจสอบ Rendo PWA v1.1.0

วันที่ตรวจสอบ: 17 กรกฎาคม 2569

## ปัญหาที่แก้

1. หน้า **รายเดือน** และ **ลงวันทำงาน** โหลดค้างหรือรับคำสั่งโหลดซ้อนกัน
2. หลังบันทึกวันทำงาน หน้าจอรอโหลดตลอดจนต้องปิดแอป
3. Auto Backup เริ่มอ่านข้อมูลทั้งฐานทันทีทุกครั้งที่เจ้าของทำรายการ ทำให้คำสั่งอื่นช้าหรือชนกันได้
4. หน้าค่าตอบแทนแสดงทุกคนต่อกันยาวและใช้งานบนมือถือยาก
5. PDF ค่าตอบแทนเปิดเป็นหน้าขาวและข้อมูลไม่ครบ
6. ไม่มีจุดตั้งค่าเพดานฐานเงินเดือนประกันสังคมที่เด่นและเข้าใจง่าย

## วิธีแก้ที่ใส่ในโค้ด

- เพิ่ม timeout สำหรับการอ่าน/เขียน Firestore และอ่านข้อความจาก cache เมื่อเครือข่ายไม่ตอบสนอง
- ป้องกันผลลัพธ์จากคำสั่งโหลดเก่ามาเขียนทับหน้าที่ผู้ใช้เปิดอยู่
- เพิ่มตัวนับคำสั่งโหลดเฉพาะหน้ารายเดือน ลงวันทำงาน และค่าตอบแทน
- หลังบันทึกวันทำงานใช้ `try/finally` เพื่อคืนสถานะปุ่มเสมอ และไม่รอ Audit/Auto Backup จนหน้าค้าง
- Auto Backup แบบ “เมื่อมีการทำรายการ” เปลี่ยนเป็นหน่วง 12 วินาทีหลังหยุดทำรายการ และห้ามสำรองหลายชุดพร้อมกัน
- หน้าค่าตอบแทนแสดงทีละคน มีแถบชื่อเลื่อนซ้าย–ขวา และปุ่มคนก่อนหน้า/คนถัดไป
- เพิ่มส่วนตั้งค่าประกันสังคมสำหรับเจ้าของ: อัตราลูกจ้าง อัตรานายจ้าง และเพดานฐานเงินเดือนสูงสุด
- PDF แสดงรายการครบถึงยอดโอนปลายเดือน รวมรายละเอียดวันทำงาน/OT และไม่แสดงยอดประกันสังคมนายจ้างหรือต้นทุนรวม
- การสร้าง PDF ย้ายเอกสารมาวาดในพื้นที่หน้าจอจริง รอรูปภาพโหลด 2 เฟรม และตรวจว่าไฟล์ Blob มีขนาดเหมาะสมก่อนดาวน์โหลดหรือแชร์
- เพิ่ม fallback สำหรับอุปกรณ์ที่แชร์ไฟล์ไม่ได้: ดาวน์โหลด PDF และคัดลอกข้อความสรุปให้
- เพิ่ม Service Worker build `1.1.0` และสั่งตรวจอัปเดตเพื่อไม่ติดไฟล์เก่า

## การตรวจสอบอัตโนมัติที่ผ่าน

- `node --check app.js` — ผ่าน
- `node --check sw.js` — ผ่าน
- ตรวจ syntax ของ `apps-script-backup.gs` ในรูปแบบ JavaScript — ผ่าน
- ตรวจ JSON: `manifest.webmanifest`, `version.json`, `firebase.json`, `.firebaserc`, `firestore.indexes.json` — ผ่าน
- ตรวจจำนวนวงเล็บ CSS `{}` เท่ากัน — ผ่าน
- ตรวจว่าไม่มี function ชื่อซ้ำใน `app.js` — ผ่าน
- Browser smoke test ด้วย Chromium/Playwright และข้อมูล Firebase จำลอง:
  - เปิดหน้ารายเดือนและแสดงตาราง/ชื่อพนักงาน — ผ่าน
  - เปิดหน้าลงวันทำงานและแสดงฟอร์ม/สรุป — ผ่าน
  - บันทึกวันทำงานแล้วปุ่มกลับมาใช้งานและขึ้นข้อความสำเร็จ — ผ่าน
  - เปิดหน้าค่าตอบแทน แสดงพนักงานเพียง 1 คนต่อครั้ง และเลื่อนไปคนถัดไปได้ — ผ่าน
  - บันทึกค่าตอบแทนรายคน — ผ่าน
  - พบช่องเพดานประกันสังคม `ssMaxBase` สำหรับเจ้าของ — ผ่าน
  - ตรวจข้อความ PDF มีเงินเดือน/ค่าจ้าง, OT, โบนัส, หักเงิน, เบิกล่วงหน้า, ประกันสังคมลูกจ้าง และยอดโอนปลายเดือน — ผ่าน
- ทดสอบวาดรายงานภาษาไทยเป็น PDF ด้วย Chromium:
  - ได้ไฟล์ PDF 1 หน้า ขนาดประมาณ 674 KB — ผ่าน
  - PDF ไม่เป็นหน้าขาว — ผ่าน
  - ดึงข้อความภาษาไทยจาก PDF ได้ — ผ่าน
  - โลโก้ ตาราง และยอดโอนแสดงในภาพตรวจสอบ — ผ่าน

## สิ่งที่ยังต้องทดสอบกับระบบจริงของร้าน

สภาพแวดล้อมสร้างไฟล์ไม่มีสิทธิ์เข้า Firebase, Google Drive และบัญชี LINE ของร้าน จึงไม่สามารถยืนยัน End-to-End กับข้อมูลจริงได้ ต้องทดสอบหลังอัปโหลดดังนี้:

1. Login ด้วยบัญชีทั้ง 8 ระดับ และตรวจสิทธิ์ทุกหน้า
2. ลงวันทำงานจริง 2–3 คนพร้อมกันจากหลายเครื่อง
3. เปิดหน้ารายเดือนจากทุกระดับผู้ใช้
4. ตั้ง Auto Backup แล้วตรวจว่าไฟล์ขึ้นใน `Rendo_Backups`
5. สร้าง PDF บน Android Chrome และ iPhone Safari
6. กด **แชร์ PDF เข้า LINE** บนมือถือที่ติดตั้ง LINE แล้วเลือกห้องแชตจริง
7. ตรวจอัตราและเพดานประกันสังคมให้ตรงกฎหมายที่ใช้ในเดือนจ่ายจริง
8. ตรวจว่า Firestore Indexes ทั้ง 2 รายการเป็นสถานะ **Enabled**

หมายเหตุ: ซอฟต์แวร์ที่เชื่อมต่อเครือข่ายไม่สามารถรับประกันว่าไม่มีข้อผิดพลาดได้ทุกกรณี แต่เวอร์ชันนี้เพิ่ม timeout, การคืนสถานะปุ่ม, การแจ้งข้อผิดพลาด และการป้องกันงานซ้อน เพื่อไม่ให้ผู้ใช้ต้องปิดแอปเมื่อเครือข่ายหรือ Firebase ตอบสนองช้า
