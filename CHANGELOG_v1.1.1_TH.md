# Rendo PWA v1.1.1 — รายการแก้ไข

## ปัญหาหลักที่แก้

- แก้หน้าลงทำงานค้างทั้งหน้าในเขตเวลาไทย สาเหตุเดิมคือฟังก์ชันเพิ่มวันแปลงวันผ่าน UTC ทำให้วันที่ไม่เดินหน้าและวนลูปไม่จบ
- เปลี่ยนการตรวจวันที่ขาดเป็นวนตามเลขวันที่ 1–31 พร้อมขอบเขตตายตัว จึงไม่มีโอกาสวนลูปค้าง
- แยกตัวนับการโหลดฟอร์มและรายการลงทำงาน ไม่ใช้ตัวเดียวกัน จึงไม่เหลือสถานะล็อกค้างเมื่อกดเร็วหรือเปลี่ยนวัน/คน/เดือน
- ระหว่างโหลดข้อมูลเดิม ปิดเฉพาะปุ่มบันทึก ไม่ล็อกทั้งฟอร์ม และยังเปลี่ยนหน้าหรือกดเมนูได้
- ป้องกันข้อมูลที่โหลดช้ามาทับข้อมูลใหม่เมื่อเปลี่ยนหน้าเร็ว

## ยอดขาย

- ช่องตัวเลขที่เริ่มต้นเป็น 0 จะเลือกเลข 0 ให้อัตโนมัติเมื่อแตะ และพิมพ์เลขใหม่แทนได้ทันที
- หากปล่อยช่องว่าง ระบบจะคืนค่าเป็น 0 ตอนออกจากช่อง
- เงินสดเปิดกะยังดึงจากเงินสดปิดกะวันเปิดร้านก่อนหน้า หากเมื่อวานหยุดจะย้อนค้นหารายการก่อนหน้าได้สูงสุด 400 รายการ และยังแก้ไขเองได้
- เพิ่มระบบยกเลิกผลโหลดเก่าและ timeout ในหน้าขาย ป้องกันเปลี่ยนหน้าเร็วแล้วข้อมูลเก่ากลับมาทับหน้าใหม่

## ความเร็วและความลื่น

- เพิ่ม cache ระยะสั้น 10 วินาทีสำหรับข้อมูลรายเดือน ลดการยิง Firestore ซ้ำเมื่อสลับหน้าไปมา
- ล้าง cache ทันทีเมื่อมีการบันทึกข้อมูล เพื่อไม่ให้เห็นข้อมูลเก่า
- ปุ่มเมนูหน้าปัจจุบันถูกปิดไว้ ป้องกันกดหน้าเดิมซ้ำจนเกิดการโหลดซ้อน
- Service Worker เปลี่ยนเป็น build 1.1.1 เพื่อบังคับรับไฟล์แก้ไขล่าสุด
