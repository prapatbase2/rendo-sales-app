# Rendo PWA v1.2.0 — รายการแก้ไข

## ลงวันทำงาน

- แก้ race condition ที่ใช้ตัวนับคำสั่งเดียวกันระหว่างโหลดแบบฟอร์มและโหลดรายการ ซึ่งเป็นต้นเหตุให้ข้อความ “กำลังโหลด” ค้าง
- แยก `attendanceFormLoadSeq` และ `attendanceResultLoadSeq`
- โหลดแบบฟอร์มและรายการแบบอิสระ ไม่บล็อกการเปิดหน้า
- เปลี่ยนวันที่/พนักงานระหว่างโหลดได้ โดยผลลัพธ์เก่าจะไม่เขียนทับหน้าปัจจุบัน
- หาก Firestore ช้าหรือผิดพลาด ระบบจะแสดงข้อความและปุ่ม “ลองใหม่” แทนการค้าง
- หลังบันทึก ล้าง cache เฉพาะ attendance และรีเฟรชข้อมูลโดยไม่รอ Audit/Auto Backup
- ลดการพึ่งพา Composite Index: โหลดตามเดือนแล้วกรองพนักงานในเครื่อง

## ยอดขายและช่องตัวเลข

- ค่าเริ่มต้นยังเป็น 0 แต่เมื่อโฟกัสช่องที่เป็น 0 ระบบล้างให้พิมพ์เลขใหม่ได้ทันที
- หากออกจากช่องโดยยังว่าง ระบบคืนค่าเป็น 0 อัตโนมัติ
- ใช้พฤติกรรมเดียวกันกับช่องจำนวนเงินที่แก้ไขได้ในหน้าหลักอื่น ๆ
- เงินสดเปิดกะดึงจากยอดปิดกะล่าสุดของวันที่ร้านเปิดก่อนหน้า ค้นย้อนหลังเป็นชุดได้สูงสุด 12 เดือน และแก้เองได้
- การโหลดข้อมูลยอดขายจริง/ชั่วคราวมี timeout และยกเลิกผลลัพธ์เก่าเมื่อเปลี่ยนหน้า

## ความลื่นไหลและความเสถียร

- ทุกครั้งที่เปลี่ยนหน้า จะยกเลิกผลลัพธ์ async ของหน้าก่อนหน้าโดยใช้ `navRequestId` และตัวนับเฉพาะหน้า
- เพิ่ม cache อายุสั้นสำหรับ query รายเดือน/รายผู้ใช้ พร้อม invalidation หลังเขียนข้อมูล
- เพิ่ม timeout สำหรับ Firestore read/write และ Audit
- หน้า Dashboard, รายเดือน, เบิกล่วงหน้า, ค่าตอบแทน, รายจ่าย และประวัติ แสดงโครงหน้าก่อน แล้วโหลดข้อมูลเบื้องหลัง
- Auto Backup แบบทำตามรายการหน่วงเวลาและไม่ทำซ้อน
- อัปเดต Service Worker เป็น build 1.2.0 เพื่อไม่ติดไฟล์เก่า

## ฟังก์ชันเดิมที่คงไว้

- หน้าค่าตอบแทนแสดงทีละคนและเลื่อนชื่อซ้าย–ขวา
- เจ้าของกำหนดอัตราและเพดานฐานเงินเดือนประกันสังคมได้
- PDF แสดงรายการครบถึงยอดที่ต้องโอนปลายเดือน
- แชร์ PDF ผ่านหน้าต่างแชร์ของมือถือเพื่อเลือก LINE; อุปกรณ์ที่ไม่รองรับจะดาวน์โหลดไฟล์และคัดลอกข้อความสรุป
