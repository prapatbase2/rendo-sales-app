# คู่มือใช้งาน Rendo v1.0

## เข้าสู่ระบบ

1. กรอกรหัสผู้ใช้
2. กรอก PIN 4 หลัก
3. กดแสดง/ซ่อนหากต้องการตรวจตัวเลข
4. เลือกจำรหัสผู้ใช้และ PIN เฉพาะเครื่องส่วนตัว
5. กดเข้าสู่ระบบ

กรอกผิด 5 ครั้ง แอปพัก 60 วินาทีในเครื่องนั้น ปุ่มล้างข้อมูลที่จำไว้จะลบ ciphertext ออกจากเครื่อง

## ระดับผู้ใช้

- เจ้าของ: ทุกหน้า สร้าง/เปลี่ยน/ปิดผู้ใช้ ดู PIN ตั้งค่า Backup และ Restore
- ผู้จัดการ: Dashboard, เช็คชื่อ, ยอดขาย, รายเดือน, เบิก, ค่าตอบแทน, รายจ่าย, ประวัติ และผู้ใช้ตามข้อจำกัด
- หัวหน้า: เช็คชื่อพนักงาน ยอดขาย รายเดือน และสร้าง 5 ระดับล่าง
- ครัวหน้าร้าน/หลังบ้าน/หน้าร้าน/เวียน/รายวัน: หน้าเฉพาะงานของตนตามสิทธิ์

## ผู้ใช้งาน

1. กดเมนูผู้ใช้งาน
2. กรอกชื่อ รหัสผู้ใช้ PIN วันที่เริ่มงาน และระดับ
3. กดสร้างบัญชี
4. เจ้าของกดดู PIN ได้หลัง re-authenticate
5. เจ้าของ/ผู้จัดการกดเงินเดือน/ธนาคารเพื่อบันทึกข้อมูล
6. ปิดใช้งานแทนการลบจริง ข้อมูลเดือนเก่ายังอ้าง UID เดิม

ผู้จัดการสร้าง manager ได้ แต่แก้/ลบ owner, manager ที่มีอยู่ หรือตัวเองไม่ได้ หัวหน้าสร้างได้เฉพาะ 5 ระดับล่าง

## ลงทำงาน / เช็คชื่อ

พนักงานระดับล่างลงเฉพาะของตัวเอง เลือกวันที่และสถานะ

- ครัว/หน้าร้าน: ทั้งวัน หยุด ลาพักผ่อน ลาป่วย ลากิจ อื่น ๆ และ OT 22:00–24:00
- พนักงานเวียน: เลือก Rendo หรือ Love Matcha ถ้า Love Matcha ทำ OT Rendo ได้ 18:00–24:00 ถ้า Rendo เริ่ม 22:00
- รายวัน: ทั้งวัน รายชั่วโมง 11:00–24:00 หรือหยุด

ลาป่วย ลากิจ และอื่น ๆ ต้องมีสาเหตุ เวลาสิ้นสุดต้องมากกว่าเวลาเริ่ม 24:00 เก็บเป็น 1440 นาที

แถบเตือนวันที่ขาดตรวจตั้งแต่ต้นเดือนถึงเมื่อวาน โดยไม่เตือนก่อนวันเริ่มงาน

## ยอดขาย

1. เลือกวันที่
2. เลือกเปิดร้านหรือหยุดร้าน
3. กรอกอาหาร เครื่องดื่ม ส่วนลด เงินสด เงินโอน และเบียร์
4. ตรวจผลต่างรายได้กับเงินสด+โอน
5. กรอกเงินเปิดกะ รายจ่าย เงินให้เจ้าของ และเงินปิดกะ
6. ตรวจสมการเงินสด
7. กดบันทึกชั่วคราวหรือบันทึกจริง

Draft มีป้ายชัดเจนและไม่เข้า Dashboard/รายเดือน/โบนัส/ค่าตอบแทน/รายจ่าย

เงินเปิดกะแนะนำจากวันเปิดร้านล่าสุด ผู้ใช้แก้ได้และระบบเก็บ manual override ถ้าแก้ยอดปิดกะย้อนหลัง ระบบทำเครื่องหมายวันถัดไปให้ตรวจเงินเปิดกะ

## รายเดือน

เลือกเดือนเพื่อดูทุกวัน รายชื่อทำงานทั้งวันแยกครัว/หน้าร้าน รายชั่วโมงและ OT ไม่อยู่ในรายชื่อทำงานทั้งวัน

พนักงานระดับล่างดูเดือนปัจจุบันและย้อนหลัง 1 เดือน เจ้าของ/ผู้จัดการ/หัวหน้าดูย้อนหลังได้ตามข้อมูลที่มี

## เบิกเงินล่วงหน้า

เจ้าของเปิดสิทธิ์รายคน ผู้ที่ได้รับสิทธิ์เห็นรายการของตน เจ้าของ/ผู้จัดการเห็นทุกคน เพิ่ม แก้ และ soft-delete ได้ รายการถูกหักจากยอดโอนเดือนเดียวกันโดย calculation กลาง

## ค่าตอบแทน

1. เลือกเดือน
2. ตรวจเงินเดือน/ธนาคารในหน้าผู้ใช้
3. ตรวจวันทำงานและยอดขาย Final
4. กรอกเพิ่มอื่น ๆ OT นอกเวลา และรายการหักพร้อมรายละเอียด
5. รายวันทำเครื่องหมายจ่ายให้แล้วรายรายการ
6. ตรวจโบนัส เงินเบิก และประกันสังคม
7. Finalize เพื่อเก็บ snapshot

การเปิดเดือน Finalized ต้องเป็นเจ้าของและกรอก PIN

ก่อนสร้าง PDF ต้องทำเครื่องหมายตรวจเลขบัญชี PDF แสดงถึงยอดที่ต้องโอนปลายเดือน ไม่แสดงประกันสังคมนายจ้างหรือต้นทุนร้าน

## ลงรายจ่าย

แบบฟอร์มเพิ่มอยู่ด้านบนก่อนตาราง

- รายจ่ายประจำเป็นแม่แบบและ snapshot ไปแต่ละเดือน
- รายจ่ายอื่นมีชื่อ วันที่ จำนวน หมายเหตุ
- ปุ่มขึ้น/ลงสลับ sortOrder โดยใช้ stable ID
- ต้นทุนค่าตอบแทนดึงจาก compensation ที่ Finalized และรวมประกันสังคมนายจ้าง
- รายจ่ายระหว่างกะไม่ถูกนับซ้ำใน owner expenses

## Dashboard

เลือกเดือนหรือช่วงวัน ตัวเลขใช้เฉพาะยอด Final

สูตร:

`รายได้หลังหักค่าใช้จ่าย = รายได้รวม × อัตราส่วนหลังต้นทุนสินค้าโดยประมาณ − รายจ่ายระหว่างกะ − รายจ่ายหน้าลงรายจ่าย`

ค่าเริ่มต้น 40% หมายถึงสัดส่วนรายได้หลังหักต้นทุนสินค้าโดยประมาณ ไม่ใช่ภาษีหรือกำไรสุทธิทางบัญชี

## ประวัติ

เจ้าของ/ผู้จัดการดู actor เวลา action area target และ Before/After เจ้าของซ่อนรายการจากผู้จัดการได้ รายการยังอยู่และเจ้าของแสดงกลับได้ ไม่มีปุ่มลบ Audit Log จริง

## ตั้งค่า

เจ้าของปรับชื่อ สี ขนาดตัวอักษร โลโก้ ลำดับเมนู อัตรากำไร OT เรทรายวัน โบนัส ประกันสังคม กฎพนักงานเวียน และกฎหมายเหตุยอดรับ

กดคืนค่าเริ่มต้นเมื่อต้องการกลับค่ามาตรฐาน การ Finalize ค่าตอบแทนเก็บ snapshot จึงไม่เปลี่ยนเดือนเก่า

## Offline และอัปเดต

Offline อ่านข้อมูลที่เคยโหลดผ่าน Firestore local cache และเปิด app shell ได้ ปุ่มเขียนถูกปิด ไม่มี offline write queue

เมื่อมี Service Worker ใหม่ แถบอัปเดตจะปรากฏ ผู้ใช้กดโหลดเวอร์ชันใหม่เมื่อไม่กำลังกรอกข้อมูล
